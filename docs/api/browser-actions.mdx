# Browser Actions API

This page documents all available browser actions that can be performed through the browser-use library. Each action is implemented as a tool that the AI agent can use to interact with web pages.

## Navigation Actions

### navigate (GoToUrlAction)

Navigate to a URL in the current tab or open a new tab.

<ParamField path="url" type="string" required>
  The URL to navigate to (must include protocol: http:// or https://)
</ParamField>

<ParamField path="new_tab" type="boolean" default="false">
  Set to true to open the URL in a new tab instead of the current tab
</ParamField>

<CodeGroup>
```python Python
from browser_use.tools.views import GoToUrlAction

action = GoToUrlAction(
    url="https://example.com",
    new_tab=False
)
```

```json MCP Tool Call
{
  "name": "browser_navigate",
  "arguments": {
    "url": "https://example.com",
    "new_tab": false
  }
}
```
</CodeGroup>

**Returns:**
```json
{
  "success": true,
  "action": "navigate",
  "url": "https://example.com",
  "tab_id": "1234",
  "loading_time_ms": 1250
}
```

### go_back (NoParamsAction)

Navigate back to the previous page in browser history.

<CodeGroup>
```python Python
from browser_use.tools.views import NoParamsAction

action = NoParamsAction()
```

```json MCP Tool Call
{
  "name": "browser_go_back",
  "arguments": {}
}
```
</CodeGroup>

## Element Interaction Actions

### click (ClickElementAction)

Click on an element by its index from the current page state.

<ParamField path="index" type="integer" required>
  Index of the element to click (must be ≥ 1, from browser_get_state results)
</ParamField>

<ParamField path="while_holding_ctrl" type="boolean" default="null">
  Set to true to perform Ctrl+Click (opens links in new background tab)
</ParamField>

<CodeGroup>
```python Python
from browser_use.tools.views import ClickElementAction

action = ClickElementAction(
    index=5,
    while_holding_ctrl=True
)
```

```json MCP Tool Call
{
  "name": "browser_click",
  "arguments": {
    "index": 5,
    "while_holding_ctrl": true
  }
}
```
</CodeGroup>

**Returns:**
```json
{
  "success": true,
  "action": "click",
  "element_index": 5,
  "element_text": "Submit Button",
  "new_url": "https://example.com/submitted",
  "execution_time_ms": 150
}
```

### type (InputTextAction)

Type text into an input field or any focusable element.

<ParamField path="index" type="integer" required>
  Index of the element to input text into (0 for page-level typing, ≥1 for specific elements)
</ParamField>

<ParamField path="text" type="string" required>
  The text to type into the element
</ParamField>

<ParamField path="clear_existing" type="boolean" default="true">
  Set to true to clear existing text first, false to append to existing text
</ParamField>

<CodeGroup>
```python Python
from browser_use.tools.views import InputTextAction

action = InputTextAction(
    index=3,
    text="user@example.com",
    clear_existing=True
)
```

```json MCP Tool Call
{
  "name": "browser_type",
  "arguments": {
    "index": 3,
    "text": "user@example.com",
    "clear_existing": true
  }
}
```
</CodeGroup>

## Scrolling Actions

### scroll (ScrollAction)

Scroll the page or a specific scrollable element.

<ParamField path="down" type="boolean" required>
  Direction to scroll (true for down, false for up)
</ParamField>

<ParamField path="num_pages" type="number" required>
  Number of viewport heights to scroll (0.5 = half page, 1.0 = full page, etc.)
</ParamField>

<ParamField path="frame_element_index" type="integer" default="null">
  Optional element index to find scroll container for (for scrolling within specific elements)
</ParamField>

<CodeGroup>
```python Python
from browser_use.tools.views import ScrollAction

action = ScrollAction(
    down=True,
    num_pages=1.5,
    frame_element_index=None
)
```

```json MCP Tool Call
{
  "name": "browser_scroll",
  "arguments": {
    "down": true,
    "num_pages": 1.5
  }
}
```
</CodeGroup>

## Error Handling

All actions can return error responses in the following format:

```json
{
  "success": false,
  "error": {
    "code": "ELEMENT_NOT_FOUND",
    "message": "Element with index 5 not found on current page",
    "action": "click",
    "details": {
      "element_index": 5,
      "page_url": "https://example.com",
      "total_elements": 4
    }
  }
}
```

### Common Error Codes

- `ELEMENT_NOT_FOUND` - Element index does not exist
- `ELEMENT_NOT_CLICKABLE` - Element cannot be clicked
- `NAVIGATION_FAILED` - URL navigation failed
- `TIMEOUT_ERROR` - Operation timed out
- `BROWSER_ERROR` - General browser operation error
- `INVALID_INPUT` - Invalid action parameters

## Best Practices

1. **Always call `get_page_state` first** to get current element indices
2. **Use descriptive extraction queries** for better AI content extraction results
3. **Handle errors gracefully** and retry operations when appropriate
4. **Use timeouts** for operations that might hang
5. **Optimize scroll operations** by using appropriate page multipliers
6. **Use new_tab=true** for navigation when you need to preserve the current page
